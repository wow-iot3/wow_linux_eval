 # WOW Linux Eval 工程设计文档

---

## 1. 项目概述

### 1.1 项目名称

**WOW Linux Eval** —— 面向嵌入式 Linux 的物联网（IoT）基础平台框架

### 1.2 项目定位

本项目是一套 **C 语言编写的嵌入式 Linux 基础组件库**，采用分层架构设计，为物联网终端设备提供从硬件抽象、操作系统封装、通用数据结构、配置管理、通信协议到应用服务的完整软件栈。项目输出为动态共享库（`libwowbase.so、libwowhal.so、libwowiot.so`），供上层应用链接调用。

### 1.3 技术栈

| 类别 | 技术选型 |
|------|----------|
| 编程语言 | C (C99) |
| 目标平台 | 嵌入式 Linux（已适配 Ubuntu 20.04） |
| 构建系统 | 自研 mbuild（基于 GNU Make + Shell） |
| 第三方库 | libev 4.33（事件驱动）、mbedTLS 2.13.1（加密） |
| 单元测试 | greatest 框架 |
| 配置格式 | INI / JSON / XML |
| 数据库 | SQLite3（内嵌源码） |

## 3. 构建系统设计 (`build/`)

### 3.1 构建系统概述

项目采用自研的 **mbuild** 构建系统，基于 GNU Make 与 Shell 脚本实现，支持多种构建目标类型。

### 3.2 构建核心文件 (`build/core/`)

| 文件 | 职责 |
|------|------|
| `main.mk` | 构建系统主入口，定义全局规则 |
| `clear_vars.mk` | 清除模块局部变量，实现模块隔离 |
| `build_executable.mk` | 可执行文件构建规则 |
| `build_library.mk` | 动态/静态库构建规则 |
| `build_fromsrc.mk` | 从源码编译第三方库 |
| `build_fromtar.mk` | 从压缩包解包编译第三方库 |
| `build_qtproject.mk` | Qt 项目构建支持 |
| `build_resource.mk` | 资源文件打包规则 |
| `build_tools.mk` | 工具程序构建规则 |
| `dep_file.mk` | 头文件依赖自动生成 |
| `mbuild-core.sh` | 构建核心 Shell 脚本 |

### 3.3 环境配置

| 文件 | 职责 |
|------|------|
| `envsetup.sh` | 设置交叉编译工具链、目标平台等环境变量 |
| `profile.sh` | 运行时环境配置（`LD_LIBRARY_PATH` 等） |

### 3.4 模板系统 (`build/templates/`)

提供 `.in` 模板文件，支持快速生成新模块的 Makefile 骨架：

- `target-exec.mk.in` → 可执行程序模块
- `target-lib.mk.in` → 库模块
- `target-qt.mk.in` → Qt 模块
- `target-res.mk.in` → 资源模块
- `from-src.mk.in` / `from-tar.mk.in` → 第三方库模块
- `CMakeLists.txt.in` → CMake 兼容支持

### 3.5 构建流程

```
envsetup.sh (设置环境)
     │
     ▼
main.mk (主控)
     │
     ├── clear_vars.mk (清理变量)
     ├── 各模块 .mk 文件 (wow_base_src.mk, wow_iot_src.mk, ...)
     │       │
     │       ├── build_library.mk (编译为 libwow.so)
     │       └── build_executable.mk (编译测试程序)
     │
     ├── 3rdparty/*.mk (编译第三方库)
     │       ├── libev.mk → build_fromtar.mk
     │       └── libmbedtls.mk → build_fromtar.mk
     │
     └── dep_file.mk (依赖管理)
```

### 5.2 硬件抽象层 `wow_hal`

> **定位**：隔离硬件差异，为上层提供统一的硬件访问接口。

#### 5.2.1 模块结构

```
wow_hal/
├── inc/
│   ├── wow_hal_errno.h      # HAL 层错误码
│   ├── wow_hal_init.h       # HAL 初始化接口
│   └── wow_usart.h          # 串口通信接口
├── src/
│   ├── hal_dev_conf.h       # 设备配置（串口设备节点等）
│   ├── hal_env.h            # HAL 运行环境定义
│   ├── wow_hal_init.c       # HAL 初始化实现
│   ├── wow_hal_internal.h   # HAL 内部头文件
│   ├── board/
│   │   └── ubuntu-2004.c    # Ubuntu 20.04 板级支持包 (BSP)
│   └── usart/
│       └── wow_usart.c      # 串口驱动实现
└── test/
    └── suit_usart.c         # 串口单元测试
```

#### 5.2.2 设计要点

- **板级支持包 (BSP) 模式**：通过 `board/` 目录管理不同硬件平台的适配代码，切换平台只需新增对应 BSP 文件
- **设备配置分离**：`hal_dev_conf.h` 集中管理设备节点路径等平台相关参数
- **当前支持**：串口（USART），可扩展 SPI、I2C、GPIO 等

##### 5.3.2.10 protocol — 工业通信协议栈 ⭐

这是本项目的 **核心业务模块**，实现了工业物联网领域主流通信协议：

```
protocol/
├── wow_proto_data.h/c       # 协议数据模型（点表/遥测/遥信/遥控）
├── wow_proto_port.h/c       # 协议端口抽象（串口/TCP 传输层适配）
│
├── modbus/                  # Modbus 协议族
│   ├── wow_modbus_x.h       # Modbus 通用定义
│   ├── wow_modbus_m.h/c     # Modbus 主站（Master）
│   ├── wow_modbus_s.h/c     # Modbus 从站（Slave）
│   ├── modbus_rtu.c         # Modbus RTU 帧编解码
│   ├── modbus_tcp.c         # Modbus TCP 帧编解码
│   └── modbus_internal.h    # 内部数据结构
│
└── iec10x/                  # IEC 60870-5 协议族
    ├── wow_iec10x.h          # IEC10x 通用定义
    ├── wow_iec101m.h/c       # IEC 101 主站
    ├── wow_iec101s.h/c       # IEC 101 从站
    ├── wow_iec104m.h/c       # IEC 104 主站
    ├── wow_iec104s.h/c       # IEC 104 从站
    ├── iec101_base.h         # IEC 101 基础帧定义
    ├── iec104_base.h         # IEC 104 基础帧定义
    └── iec10x_base.h         # IEC 10x 公共帧定义
```

**协议支持矩阵**：

| 协议 | 传输层 | 主站 (Master) | 从站 (Slave) |
|------|--------|:---:|:---:|
| Modbus RTU | 串口 (RS485) | ✅ | ✅ |
| Modbus TCP | TCP/IP | ✅ | ✅ |
| IEC 60870-5-101 | 串口 | ✅ | ✅ |

##### 5.3.2.11 shmcache — 共享内存缓存


## 9. 关键设计模式与原则

### 9.1 设计模式应用

| 模式 | 应用场景 |
|------|----------|
| **分层架构** | HAL → Base → IoT → App 四层解耦 |
| **策略模式** | 数据库抽象层（可替换 SQLite3 为其他引擎） |
| **单例模式** | `wow_singleton` 全局唯一对象管理 |
| **迭代器模式** | `wow_iterator` 容器统一遍历 |
| **工厂模式** | `wow_element` 泛型元素创建 |
| **观察者模式** | 协议栈事件回调机制 |
| **模板方法** | 构建系统的 `.mk.in` 模板文件 |

### 9.2 编码规范要点

- **前缀命名**：所有对外符号以 `wow_` 开头
- **模块隔离**：`_internal.h` 仅模块内部使用，不对外暴露
- **头文件分离**：`inc/` 为公开 API，`src/` 内的 `.h` 为私有
- **错误处理**：每个层级有独立错误码定义（`wow_errno.h`、`wow_hal_errno.h`、`wow_iot_errno.h`）
- **初始化机制**：每个模块提供 `*_init.h/c`，支持模块独立初始化

## 12. 扩展性设计

### 12.1 新增硬件平台

1. 在 `wow_hal/src/board/` 下新增 BSP 文件（如 `imx6ull.c`）
2. 修改 `hal_dev_conf.h` 配置设备节点
3. 构建时选择目标 board

### 12.2 新增通信协议

1. 在 `wow_iot/inc/protocol/` 下新增协议头文件目录
2. 在 `wow_iot/src/protocol/` 下实现协议编解码
3. 复用 `wow_proto_port` 的传输层适配接口
4. 在 `wow_iot/test/protocol/` 下添加测试套件

### 12.3 新增第三方库

1. 将源码包放入 `3rdparty/`
2. 参考 `libev.mk` 编写对应 `.mk` 文件
3. 如需 `autobuild.sh` 则基于模板生成

## 13. 总结

**WOW Linux Eval** 是一套面向嵌入式 Linux 物联网场景的 **全栈 C 基础框架**，具备以下特点：

| 特性 | 描述 |
|------|------|
| 🏗️ **分层清晰** | HAL → Base → IoT → App 四层解耦，职责明确 |
| 🔌 **协议丰富** | 支持 Modbus RTU/TCP、IEC 101/104 主从站 |
| 🧩 **组件完整** | 从基础数据结构到数据库、配置、加密、网络一应俱全 |
| 🔧 **自研构建** | mbuild 系统灵活支持多种构建目标和交叉编译 |
| 🧪 **测试驱动** | 基于 greatest 的全面单元测试覆盖 |
| 📖 **文档完善** | 包含 API 说明、专题技术文档、编码规范 |
| 📦 **依赖可控** | 第三方库最小化（仅 libev + mbedTLS），源码内嵌 SQLite/cJSON/ezxml |
| 🔄 **易于扩展** | 模块化设计，新增平台/协议/功能成本低 |
